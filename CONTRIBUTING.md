# Contributing / 貢獻指南

Language: [English](#english) | [繁體中文](#繁體中文)

## English

Thanks for helping improve Codex Claude Bridge.

This project is intentionally small: Codex stays the operator, while Claude is only a read-only advisor. Contributions should preserve that boundary.

Good contribution areas:

- Installation and setup improvements.
- Clearer documentation or examples.
- Safer review prompts.
- Better secret redaction or context truncation.
- Bug fixes in the wrapper script.

Before opening a pull request:

1. Keep changes focused.
2. Run the setup or relevant command locally when possible.
3. Make sure Claude is not given write or shell permissions.
4. Explain the user-facing impact in the pull request.

For questions or workflow ideas, prefer GitHub Discussions. For reproducible bugs, open an issue.

## 繁體中文

謝謝你願意協助改進 Codex Claude Bridge。

這個專案刻意保持很小：Codex 是操作者，Claude 只是唯讀顧問。任何貢獻都應該保留這條邊界。

適合貢獻的方向：

- 安裝與 setup 改善。
- 更清楚的文件或使用範例。
- 更安全的 review prompt。
- 更好的 secret 遮蔽或 context 截斷。
- wrapper script 的 bug fix。

開 pull request 前：

1. 讓變更保持聚焦。
2. 盡可能在本機跑過 setup 或相關指令。
3. 確認沒有授予 Claude 寫入或 shell 權限。
4. 在 PR 中說明對使用者的影響。

使用問題或 workflow 想法適合開 GitHub Discussions；可重現的 bug 適合開 issue。
